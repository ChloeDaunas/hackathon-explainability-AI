# 🛡️ TalentGuard AI — Hackathon Trusted AI x HR

## Objective

An imaginary company is facing a high resignation rate. **TalentGuard AI** is a solution that helps HR management:

- **Identify** employees at risk of resignation
- **Understand** the explanatory factors (no black box)
- **Act** ethically, transparently and in compliance with GDPR

The project covers **two Trusted AI axes**:

| Axis | What we do |
|------|------------|
| **Cybersecurity** | GDPR anonymization, EU AI Act classification, input validation, exclusion of sensitive attributes |
| **Ethics** | Fairness audit (gender, ethnicity), SHAP + LIME explainability, AIF360 validation |

## Installation

**Requirements**: Python 3.10+

```bash
# 1. Clone the repo
git clone <REPO_URL>
cd talentguard-ai

# 2. Create and activate virtual environment
python -m venv venv
source venv/bin/activate        # Linux/Mac
venv\Scripts\activate           # Windows

# 3. Install dependencies
pip install -r requirements.txt
```

## Running the Pipeline

Execute the scripts in order:

```bash
# Step 1 — GDPR data anonymization (Cybersecurity axis)
python src/anonymize_data.py

# Step 2 — ML model training (full + reduced)
python src/train_model.py

# Step 3 — Fairness audit + SHAP + LIME (Ethics axis)
python src/fairness_xai.py

# Step 4 — NLP sentiment analysis
python src/nlp_pipeline.py

# Step 5 — Launch the application
streamlit run app.py
```

The application is available at `http://localhost:8501`

## Data

**Source**: [Human Resources Dataset — Kaggle (Rich Huebner & Carla Patalano)](https://www.kaggle.com/datasets/rhuebner/human-resources-data-set)

- ~311 employees (after anonymization)
- Target variable: `Termd` (0 = active, 1 = left)
- Turnover rate: ~33%
- Sensitive attributes present: `Sex`, `RaceDesc`, `GenderID`
- Enriched dataset: `HRDataset_v14_enriched.csv` — adds `ExitInterview_Feedback` and `InternalTransferRequest` columns (AI-generated text, aligned with HR data)

**Data Card — Transformations applied:**

| Technique | Columns affected | Justification |
|-----------|-----------------|---------------|
| Suppression | `Employee_Name`, `ManagerName` | Direct identifiers |
| Pseudonymization | `EmpID` | Truncated SHA-256 hash, non-reversible |
| Generalization | `DOB` → `AgeGroup` | Useful info without exact date |
| Masking | `Zip` → first 2 digits | Reduction of geographic precision |
| Model exclusion | `Sex`, `RaceDesc`, `GenderID` | Retained only for fairness audit |

## Risk simulator

The risk simulator uses a **reduced model** trained on only the most important features (importance > 0.025):

| Model | Features | Size | Inference |
|-------|----------|------|-----------|
| Full RF | 133 | ~10 MB | ~5 ms |
| Reduced RF | ~8 | ~2 MB | <1 ms |

Same prediction quality · ~5x lighter · No GPU required.
